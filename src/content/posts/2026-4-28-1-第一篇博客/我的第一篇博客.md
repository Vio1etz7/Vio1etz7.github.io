# 搭建博客的流程

1.**在github上新建一个库**，命名方式是：usename.github.io    ***注意**：username是github的用户名，不是profile里展示给别人的name*，选择公开仓库（public），因为GitHub Pages[^1] 的免费版只对公开仓库开放，如果设置为private别人就看不到你的博客了。

**建议**：建议勾选Add a README file，这样新建好的仓库不是空的，方便拉取到本地

2.**下载node.js**，访问node.js 官网 https://nodejs.org/zh-cn，  下载适合自己电脑的版本后双击运行.msi文件，傻瓜式安装就好。

验证安装：打开cmd/powershell 输入`node - v` 和 `npm -v` 输出版本号即可视为成功。

3.**安装git**，可以参考[这篇文章](https://blog.csdn.net/qq_39809160/article/details/145712755?ops_request_misc=elastic_search_misc&request_id=d695836461e7cb6896ab4f46160f423e&biz_id=0&utm_medium=distribute.pc_search_result.none-task-blog-2~all~top_positive~default-1-145712755-null-null.142^v102^pc_search_result_base4&utm_term=git%E5%AE%89%E8%A3%85&spm=1018.2226.3001.4187)

4.**安装hexo**：直接在终端执行 `npm install -g hexo-cli` 来全局安装Hexo工具

5.在**非系统盘**里创建一个文件夹作为hexo的根目录，路径最好不要有中文，cd到该目录下，并执行

`hexo init my-blog`

`cd my-blog`

`npm install`

`hexo init "文件名"`    会帮你拉取一套完整的博客模板。 

`npm install` 会安装所需的各种“零件”（依赖包）。

`hexo s`  在本地预览，类似在本地跑前端，访问 http://localhost:4000/ 就可以预览自己的博客

6.**连接github库和本地文件**

在my-blog文件夹下找到 _config.yml ，拉到底部，把deploy部分改成如下格式(注意缩进)

`deploy:
  type: git
  repo: https://github.com/你的用户名/你的用户名.github.io.git
  branch: main`

7.**安装部署插件**：

在终端运行 `npm install hexo-deployer-git --save`

8.**发布到互联网**

在终端依次输入以下命令：

`hexo clean  # 清理缓存`
`hexo g      # generate：生成静态 HTML 页面`
`hexo d      # deploy：推送到 GitHub`

做完这些就可以访问   [https://github.com/你的用户名/你的用户名.github.io.git/](https://github.com/你的用户名/你的用户名.github.io.git/)   看到你的博客了

# 发布文章的流程

1.**创建新文章**：在博客目录下打开终端，输入：`hexo new "文章名"`

Hexo 会在 `source/_posts/` 目录下自动为你创建一个名为 `文章名.md` 的文件

2.**编辑内容**：写博客，markdown格式。

3.**本地预览**(可选)：在终端运行 `hexo s` 后可以直接在  http://localhost:4000/  预览

4.**推送到互联网**：在终端执行推送三部曲：

`hexo clean` 
`hexo g `   
`hexo d`      

# 这里总结一下我用到的命令

`npm install -g hexo-cli`   下载hexo

`hexo init "文件名"` 创建文件夹并拉取一套完整的博客模板。

`npm install` 在init的文件夹下执行，会安装所需的各种“零件”（依赖包）

`hexo s`  在本地预览，类似在本地跑前端

`npm install hexo-deployer-git --save`  	安装git插件[^2]

`hexo clean    `   清理缓存
`hexo g      `         generate：生成静态 HTML 页面
`hexo d      `         deploy：推送到 GitHub

`hexo new "文章名"`  新建一篇文章

其中 

`hexo clean    `   
`hexo g      `         
`hexo d      `  

这三个命令是常用命令，他们是提交时的常用命令

[^1]: **GitHub Pages** 是 GitHub 提供的一项静态站点托管服务,它允许你直接从GitHub仓库中获取 HTML、CSS 和 JavaScript 文件，并将其发布成一个真实的网站。
[^2]: 简单来说，这个命令是给 Hexo 安装一个“快递插件”，专门负责把你本地生成的网页文件，“打包”并“邮寄”到 GitHub 的仓库里。
