我的第一篇博客

作为一个大二的软工专业学生，也有可能是为了给我的工作简历加点东西，我觉得是时候写点东西来记录我的学习和经验了。
写博客这个说法还是我从一个朋友（也是软工）那里听来的，他老师说要他们写博客（我觉得他没写），我觉得这是很好的想法，以及我的另一个朋友也经营着一个微信博客，所以，我终于开始搞了。
对于一个博客的小白，我问了gemini，他给我推荐了掘金，CSDN，知乎，博客园，hexo/hugo+github pages，对于CSDN和知乎，我觉得有些无趣，尝试了更极客（也是刚学到的词~~）的做法：hexo+github pages，跟着他的教程，我成功搭建了一个博客。
我的博客地址是：https://vio1etz7.github.io

（虽然你能看到这篇文章也就代表你已经进入我的博客了）

以下是我搭建博客的流程

1.在github上新建一个库，命名方式是：usename.github.io 注意：username是github的用户名，不是profile里的name
2.下载node.js，这一步因为我做过前端开发（vue3+typescript），所以已经安装好了
3.安装git，这一步我也和同学做过项目的协作，所以也安装好了
4.安装hexo：在终端执行 npm install -g hexo-cli

5.在非系统盘里创建一个文件夹作为hexo的根目录，路径最好不要有中文，cd到该目录下，并执行

hexo init my-blog cd my-blog npm install

hexo init "文件名" 会帮你拉取一套完整的博客模板。

npm install 会安装所需的各种“零件”（依赖包）。

hexo s 在本地预览，类似在本地跑前端

6.连接github库和本地文件

在my-blog文件夹下找到 _config.yml ，拉到底部，把deploy部分改成如下格式(注意缩进)

deploy: type: git repo: https://github.com/你的用户名/你的用户名.github.io.git branch: main

7.安装部署插件：

在终端运行 npm install hexo-deployer-git --save

8.发布到互联网

在终端依次输入以下命令：

hexo clean # 清理缓存 hexo g # generate：生成静态 HTML 页面 hexo d # deploy：推送到 GitHub

做完这些就可以访问 https://github.com/你的用户名/你的用户名.github.io.git/ 看到你的博客了

这里总结一下我用到的命令

npm install -g hexo-cli 下载hexo

hexo init "文件名" 创建文件夹并拉取一套完整的博客模板。

npm install 在init的文件夹下执行，会安装所需的各种“零件”（依赖包）

hexo s 在本地预览，类似在本地跑前端

npm install hexo-deployer-git --save 安装git插件

hexo clean 清理缓存
hexo g generate：生成静态 HTML 页面
hexo d deploy：推送到 GitHub

hexo new "文章名" 新建一篇文章

其中

hexo clean
hexo g
hexo d

这三个命令是常用命令，他们是提交时的常用命令

写在后面：写博客真不是个轻松活，我写这么一篇简陋的文章花了两个小时吧，坚持坚持。
